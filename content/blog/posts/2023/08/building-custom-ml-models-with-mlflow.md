## Introduction

In this text, we will explore the process of building a custom ML model using MLflow. Drawing inspiration from iCarros anti-fraud project, we’ll discuss how MLflow can help us to overcome specific challenges faced during model deployment.

At iCarros, we have developed a sophisticated anti-fraud model to ensure the security of our platform. As part of this project, we encountered challenges when deploying the model to a SageMaker endpoint. We encountered two specific issues:

1. Accessing predict_proba Output: By default, MLflow's log_model function logs the result of the predict method. However, in the context of our anti-fraud project, we needed access to the probability vector generated by the predict_proba method. To address this requirement, we customized the MLflow implementation by creating a custom stacking classifier, CustomStackingClassifier. This class extends the mlflow.pyfunc.PythonModel class and modifies the predict method to return the probability predictions instead of the class labels.
2. Combining XGBoost and scikit-learn Libraries: Our anti-fraud model leverages both the XGBoost and scikit-learn libraries. However, during the deployment process, we encountered difficulties when using the MLflow functions mlflow.sklearn.save_model or mlflow.xgboost.save_model. Although the model works seamlessly locally, we faced challenges with the deployed endpoint. To address this issue, we devised a workaround by leveraging MLflow's log_model function along with a custom model wrapper called StackingClassifierModelWrapper. This wrapper class extends the mlflow.pyfunc.PythonModel class and allows us to handle the loading of the custom model and its artifacts during deployment. By using this approach, we were able to successfully deploy the model and ensure its correct functioning in the production environment.

## Constructing the Custom Model

To provide a practical example, we trained the custom model using the breast cancer dataset obtained from sklearn.datasets.load_breast_cancer. Inspired by our iCarros anti-fraud project, we constructed a stacking ensemble, consisting of three XGBoost classifiers with different hyperparameters. The model was trained using scikit-learn's StackingClassifier.

The code responsible for training the model is located in the path src/train_model.py. In this file, we have implemented the key elements that effectively address the challenges mentioned earlier. Let's delve into the details of these elements to gain a better understanding of how they solve the problems at hand.

```python
import mlflow
import numpy as np

class CustomStackingClassifier(mlflow.pyfunc.PythonModel):
    def __init__(self, estimators, final_estimator):
        self.estimators = estimators
        self.final_estimator = final_estimator

    def predict(self, context, model_input):
        # Load the individual models
        xgb_models = {}
        for name, path in self.estimators.items():
            xgb_models[name] = mlflow.xgboost.load_model(path)

        # Load the final estimator
        final_estimator = mlflow.sklearn.load_model(self.final_estimator)
        
        # Make predictions
        # Individual models
        response0 = np.stack(
                             [
                              xgb_models[x].predict_proba(model_input)[:, 1] 
                              for x in xgb_models
                             ], 
                             axis=1
                            )
        # Final estimator
        predictions = final_estimator.predict_proba(response0)

        return predictions

class StackingClassifierModelWrapper(mlflow.pyfunc.PythonModel):
    def load_context(self, context):
        # Load the custom model from the context
        estimators = {
                      'xgb1': context.artifacts["estimator-xgb1"],
                      'xgb2': context.artifacts["estimator-xgb2"],
                      'xgb3': context.artifacts["estimator-xgb3"]
                     }
        final_estimator = context.artifacts["final_estimator"]

        return CustomStackingClassifier(estimators, final_estimator)

    def predict(self, context, model_input):
        model = self.load_context(context)
        return model.predict(context, model_input)
```

We created the CustomStackingClassifier class, extending the mlflow.pyfunc.PythonModel class, to handle the challenges faced. This custom model class allows us to access the predict_proba method and combines models from different libraries seamlessly. Within the CustomStackingClassifier class, we load individual models from the XGBoost library using the mlflow.xgboost.load_model function. For the final estimator, we load the model generated by scikit-learn using the mlflow.sklearn.load_model function. By leveraging these functions and customizing MLflow's implementation, we ensure compatibility and smooth integration of the stacked models.

To facilitate the loading of the custom model within the MLflow context, we introduced the StackingClassifierModelWrapper class. Extending the mlflow.pyfunc.PythonModel class, this wrapper class allows us to retrieve the necessary artifacts from the context and initialize an instance of the CustomStackingClassifier. This approach ensures that the custom stacking classifier can be seamlessly integrated into the MLflow workflow, enabling easy deployment and generation of predictions.

```python
from utils import find_project_path
import shutil
import pandas as pd
from xgboost.sklearn import XGBClassifier
from sklearn.ensemble import StackingClassifier
def train_model():
    project_path = find_project_path()
    mlflow.set_tracking_uri(f'{project_path}/mlruns')
    mlflow.set_experiment('custom-ml')
    with mlflow.start_run(run_name='train_model') as run:
        #Load dataset
        df_train = pd.read_csv(f'{project_path}/dataset/train.csv')
        X_train = df_train.drop(['target'],axis=1)
        y_train = df_train['target']
        
        #Train model
        stack0 = [
                  (
                   'xgb1', XGBClassifier(
                                         booster='gbtree', 
                                         objective='binary:logistic', 
                                         tree_method='hist', 
                                         max_depth=10, 
                                         random_state=61
                                        )
                  ),
                  (
                   'xgb2', XGBClassifier(
                                         booster='gbtree', 
                                         objective='binary:logistic', 
                                         tree_method='hist', 
                                         max_depth=8, 
                                         random_state=61
                                        )
                  ),
                  (
                   'xgb3', XGBClassifier(
                                         booster='gbtree', 
                                         objective='binary:logistic', 
                                         tree_method='hist', 
                                         max_depth=6, 
                                         random_state=61
                                        )
                  )
                 ]
        model0 = StackingClassifier(estimators=stack0, cv=3)
        model = model0.fit(X_train, y_train)
        
        #Save the artifacts
        artifacts = {}
        #Save the xgboost classifier models
        for n, m in model.named_estimators_.items():
            path = f"{project_path}/artifacts/{n}"
            artifacts[f'estimator-{n}'] = path
            mlflow.xgboost.save_model(m, path)
        #Save the final estimator (sklearn model)
        path = f"{project_path}/artifacts/final_estimator"
        artifacts['final_estimator'] = path
        mlflow.sklearn.save_model(model.final_estimator_, path)
        
        #Log the final model
        model_wrapper = StackingClassifierModelWrapper()
        mlflow.pyfunc.log_model(
                                artifact_path='custom-ml',
                                python_model=model_wrapper,
                                artifacts=artifacts,
                                code_path=[f'{project_path}/src/custom_stacking_classifier.py'],
                                conda_env=f'{project_path}/conda.yaml'
                               )
        
        #Delete the artifacts in the root of the project
        shutil.rmtree(f'{project_path}/artifacts')
        
        #Access rund_id and experiment_id
        run_id = run.info.run_uuid
        experiment_id = run.info.experiment_id
        
        #End run
        mlflow.end_run()
    
    print(F'RUN ID: {run_id}')
    print(F'EXPERIMENT ID: {experiment_id}')
    return run_id, experiment_id
```

In the train_model function, we have included the necessary code to train the custom stacking classifier model. Let's focus on the explanation of the arguments passed to the mlflow.pyfunc.log_model method during the model logging process:

- artifact_path='custom-ml': This argument specifies the path where the model artifacts will be stored within the MLflow run. In this case, the custom model artifacts will be saved under the 'custom-ml' directory.
- python_model=model_wrapper: Here, we pass the model_wrapper instance, which is an object of the StackingClassifierModelWrapper class. This model wrapper encapsulates the custom stacking classifier and provides the necessary functionality to load the model during inference.
- artifacts=artifacts: We pass the artifacts dictionary, which contains the paths to the individual XGBoost classifier models and the final scikit-learn estimator. These artifacts are crucial for the successful loading and utilization of the custom stacking classifier during inference.
- code_path=[f'{project_path}/src/custom_stacking_classifier.py']: This argument specifies the path to the source code file (custom_stacking_classifier.py) that defines the custom stacking classifier implementation. It ensures that the code is logged alongside the model artifacts, allowing for reproducibility and transparency.
- conda_env=f'{project_path}/conda.yaml': Here, we provide the path to the conda.yaml file, which contains the environment configuration needed to reproduce the Python environment required for the model. This ensures that the inference environment matches the training environment, preventing any potential inconsistencies.

By passing these arguments to the mlflow.pyfunc.log_model method, we ensure that the custom stacking classifier model, along with its associated artifacts, code, and environment configuration, is appropriately logged within the MLflow run. This comprehensive logging enables easy tracking, reproducibility, and utilization of the custom ML model during inference.
